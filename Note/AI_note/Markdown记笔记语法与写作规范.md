# Markdown 记笔记语法与写作规范

## 一、Markdown 简介

Markdown 是一种轻量级标记语言，常用于写学习笔记、技术文档、博客、项目说明等。

Markdown 文件一般以 `.md` 结尾，例如：

```text
Python学习笔记.md
C语言指针.md
数据结构-链表.md
Git常用命令.md
```

Markdown 的优点：

- 语法简单，容易上手
- 纯文本格式，文件轻量
- 适合记录代码、公式、表格和学习计划
- 可以在 VS Code、Typora、Obsidian、GitHub 等工具中使用

---

## 二、标题语法

Markdown 使用 `#` 表示标题。

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

建议笔记结构：

```markdown
# 笔记标题

## 一、学习目标

## 二、核心概念

### 1. 概念一

### 2. 概念二

## 三、代码示例

## 四、易错点

## 五、总结
```

写作规范：

- `#` 一般只用于整篇笔记的主标题
- `##` 用于主要章节
- `###` 用于章节下的小节
- 不建议标题层级跳跃，例如从 `##` 直接跳到 `#####`

---

## 三、段落与换行

普通文字直接书写即可。

```markdown
这是第一段文字。

这是第二段文字。
```

Markdown 中，**空一行表示新段落**。

如果想强制换行，可以在行尾添加两个空格，或者使用 `<br>`：

```markdown
这是第一行  
这是第二行
```

```markdown
这是第一行<br>
这是第二行
```

推荐写笔记时使用空行分段，这样结构更清楚。

## 四、加粗、斜体和删除线

### 1. 加粗

```markdown
**这是加粗文字**
```

效果：**这是加粗文字**

### 2. 斜体

```markdown
*这是斜体文字*
```

效果：*这是斜体文字*

### 3. 加粗加斜体

```markdown
***这是加粗加斜体文字***
```

效果：***这是加粗加斜体文字***

### 4. 删除线

```markdown
~~这是被删除的文字~~
```

效果：~~这是被删除的文字~~

使用建议：

```markdown
**重点：** 指针变量存储的是地址。

*注意：数组名在大多数情况下表示首元素地址。*
```

不要整段文字都加粗，加粗应该主要用于突出关键词、重点结论和易错点。

## 五、列表语法

### 1. 无序列表

推荐使用 `-` 表示无序列表。

```markdown
- Python
- C语言
- 数据结构
- 操作系统
```

### 2. 有序列表

```markdown
1. 安装 Python
2. 安装 VS Code
3. 创建项目文件夹
4. 编写代码
```

### 3. 嵌套列表

```markdown

  - C语言
  - Python
  - Java
- 计算机基础
  - 数据结构
  - 操作系统
  - 计算机网络
```

子列表一般缩进两个或四个空格。

### 4. 任务列表

```markdown
- [x] 安装 Python
- [x] 安装 VS Code
- [ ] 学习 Markdown
- [ ] 完成学习笔记
```

任务列表适合记录学习计划、每日任务和项目进度。

## 六、引用语法

使用 `>` 表示引用。

```markdown
> 这是引用内容。
```

也可以多层引用：

```markdown
> 第一层引用
>> 第二层引用
>>> 第三层引用
```

适合在笔记中标记老师强调的内容、重要结论或自己的提醒：

```markdown
> 重点：数组名不是变量，不能被赋值。
```

```markdown
> 老师上课强调：链表的核心是指针域。
```

## 七、代码语法

### 1. 行内代码

使用反引号包裹代码、命令、函数名、文件名。

```markdown
使用 `printf()` 可以输出内容。
```

适合写：

```markdown
`main()` 是程序入口函数。
`README.md` 是项目说明文件。
运行 `python main.py` 可以启动程序。
```

### 2. 多行代码块

使用三个反引号包裹代码。

````markdown
```c
#include <stdio.h>

int main() {
    printf("Hello World\n");
    return 0;
}
```
````

### 3. 指定代码语言

代码块开头可以写语言名称，用于语法高亮。

````markdown
```c
// C语言代码
```

```cpp
// C++代码
```

```python
# Python代码
```

```bash
# 命令行代码
```

```json
{
  "name": "demo"
}
```
````

写程序笔记时，建议每个代码块都写明语言。

## 十、图片语法

图片语法：

```markdown
![图片说明](图片路径)
```

例如：

```markdown
![链表示意图](./images/linked-list.png)
```

如果是网络图片：

```markdown
![Logo](https://example.com/logo.png)
```

图片说明的作用：

- 图片加载失败时显示
- 帮助别人理解图片内容
- 有利于文档可读性

## 十一、表格语法

表格适合整理概念、命令、错误和对比内容。

```markdown
| 名称 | 作用 | 备注 |
|---|---|---|
| Python | 编程语言 | 适合 AI 和自动化 |
| C语言 | 编程语言 | 适合底层和数据结构 |
| Markdown | 文档格式 | 适合写笔记 |
```

### 表格对齐

```markdown
| 左对齐 | 居中对齐 | 右对齐 |
|:---|:---:|---:|
| 内容 | 内容 | 内容 |
| 内容 | 内容 | 内容 |
```

### 学习笔记常用表格

```markdown
| 错误 | 原因 | 解决方法 |
|---|---|---|
| 忘记分号 | C语言语法要求 | 在语句末尾添加分号 |
| 变量未定义 | 使用前没有声明 | 先定义变量 |
```

## 十二、数学公式

部分 Markdown 编辑器支持 LaTeX 数学公式，例如 Typora、Obsidian、VS Code 插件和 GitHub 部分场景。

### 1. 行内公式

```markdown
一元二次方程公式为 $x=\frac{-b\pm\sqrt{b^2-4ac}}{2a}$。
```

### 2. 独立公式

```markdown
$$
x=\frac{-b\pm\sqrt{b^2-4ac}}{2a}
$$
```

常见公式：

```markdown
$$
a^2+b^2=c^2
$$
```

```markdown
$$
\sum_{i=1}^{n} i = \frac{n(n+1)}{2}
$$
```

```markdown
$$
A^{-1} = \frac{1}{|A|}A^*
$$
```

注意：不同平台对数学公式支持程度不同。

## 十三、转义字符

如果想显示 Markdown 符号本身，可以在前面加反斜杠 `\`。

```markdown
\# 这不是标题
\*\*这不是加粗\*\*
```

常见需要转义的符号：

```text
\   反斜杠
`   反引号
*   星号
_   下划线
{}  花括号
[]  方括号
()  圆括号
#   井号
+   加号
-   减号
.   英文句点
!   感叹号
```

## 十四、HTML 语法

Markdown 支持部分 HTML。

### 1. 换行

```html
第一行<br>
第二行
```

### 2. 居中

```html
<div align="center">
  <h1>标题</h1>
</div>
```

### 3. 控制图片宽度

```html
<img src="./images/demo.png" width="500">
```

### 4. 折叠内容

```html
<details>
<summary>点击展开</summary>

这里是隐藏内容。

</details>
```

使用建议：

- 普通学习笔记尽量使用 Markdown 原生语法
- 只有在需要控制图片大小、居中或折叠内容时再使用 HTML

## 二十、Markdown 写作规范

### 1. 标题层级清晰

推荐：

```markdown
# 主标题

## 一级章节

### 二级章节
```

不推荐：

```markdown
# 主标题
#### 突然四级标题
## 又回到二级标题
```

### 2. 标题前后空一行

推荐：

```markdown
## 安装方法

下面是安装步骤。
```

### 3. 列表符号保持统一

推荐：

```markdown
- 安装 Python
- 配置环境变量
- 测试安装结果
```

不推荐：

```markdown
- 安装 Python
* 配置环境变量
+ 测试安装结果
```

### 4. 代码块尽量标明语言

推荐：

````markdown
```python
print("Hello")
```
````

### 5. 文件路径、命令、函数名使用行内代码

推荐：

```markdown
请打开 `README.md` 文件，然后运行 `python main.py`。
```

## 二十二、总结

用 Markdown 记笔记时，最重要的是结构清晰。

建议固定使用下面的笔记结构：

```markdown
# 笔记标题

## 学习目标

## 核心概念

## 代码示例

## 易错点

## 总结
```

优先掌握：

- 标题
- 列表
- 代码块
- 表格
- 链接
- 图片
- 任务列表
- 引用
- 加粗与行内代码

掌握这些后，就已经可以写出非常清晰、规范的学习笔记。
